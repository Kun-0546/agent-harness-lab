# Sample Workspace — Agent Harness Lab Complete Product Flow

This workspace demonstrates the full Agent Harness Lab workflow end-to-end.
**Fully local, deterministic, offline, zero API keys.**

## Story

You're an AI PM evaluating an FAQ-bot whose answers are verbose. You want to
compare two harness variants on the same questions:

- **V1 (baseline)** — uses the runtime's default verbose system prompt
- **V2** — uses the `concise-prompt@0.1.0` harness package, which installs
  a stricter, more concise system prompt

After running, the compare report should show V2 produced different (more
concise) responses, scored differently by the stub grader.

## Prerequisites

- Python 3.10+ on PATH as `python` (Windows: `py` works too, see
  "Troubleshooting" below)
- Agent Harness Lab installed (`pip install -e .` from the repo root)

## 5-step recipe

```bash
# (from the repo root)
cd examples/sample-workspace

# Step 1 — optional pre-run readiness check
ahl probe 001
#   expected: V1 status=ok, V2 status=ok, exit 0
#   artifact: experiments/001-faq-conciseness/probe-results/probe-<timestamp>/

# Step 2 — run the experiment (2 variants × 2 cases = 4 conversations)
ahl run 001
#   expected: 4 records in experiments/001-faq-conciseness/results/run-*.json
#   snapshots written to experiments/001-faq-conciseness/results/snapshots/

# Step 3 — score using stub grader (no LLM, no API keys)
ahl score 001
#   expected: results/score-*.json with top-level `evidence` block
#   V1 evidence = strong (materialized local_path)
#   V2 evidence = strong (materialized local_path + package fully fingerprinted)

# Step 4 — compare
ahl compare 001
#   expected: results/compare-*.md with `## Evidence` section + version totals;
#   V1 vs V2 totals differ because the package changed agent behavior

# Step 5 — review (optional; read-only summary including last probe)
ahl review 001
```

## What you should see

After Step 4, open the latest `compare-*.md`:

- `## Evidence` table showing both V1 and V2 at level `strong`
- V2's reasons mention `concise-prompt@0.1.0 fully fingerprinted`
- 版本总分 section showing V1 and V2 at different totals (the
  `concise-prompt` package altered V2's response prefix, which the stub
  grader hashes into a different score)

| Concept | Where it shows up |

## Troubleshooting

- **`python: command not found`** on Windows: edit `harnesses/V1.md` and
  `harness-packages/concise-prompt/0.1.0/manifest.md`, replace
  `python agent.py` with `py agent.py`. (Or add `python` to PATH via the
  Python installer's "Add to PATH" option.)
- **`runtime_source 'local-tiny' not in runtime-sources.md`**: make sure
  you ran `cd examples/sample-workspace` first — relative paths in
  `runtime-sources.md` are resolved from your current working directory.
- **Scores look "fake"**: yes — the default `stub_grader` is a deterministic
  hash of `(dimension, agent_text)`. For real LLM-judged scores, set the
  `AHL_JUDGE_BASE_URL` / `AHL_JUDGE_MODEL` / `AHL_JUDGE_API_KEY` env vars
  and run `ahl score 001 --llm`. (Out of scope for this sample.)

## What's NOT generated by `git`

The following are written to disk when you run the sample but are
**ignored by `.gitignore`** so they don't pollute the repo:

- `experiments/001-faq-conciseness/results/`
- `experiments/001-faq-conciseness/probe-results/`
- `experiments/001-faq-conciseness/sandbox/`
- `experiments/001-faq-conciseness/materials/runtime-evidence.md`

If you want to start fresh, just delete those directories (or run the
sample in a different working copy).
