## 已经否掉的——别再提

- **「变体」这个词** → 用「版本」/ V1、V2、V3。
- **「赢家 / 赢家链 / 自动晋级」** → 没这套机制。每个实验是一份可复现记录，不产生「赢家」。
- **step 四 / 闭环 / 应用到生产 agent** → 本版不做，推迟。工具只读 agent、不写。
- **照搬 AutoResearch** → 它是参考、不是模板。我们跟它共享「两个角色」的结构，但玩法不同（我们版本对比，它单文件一路拧）。它是 2 个角色，不是 3 个。

- **design-v0.1 的五元素模型** → 已重组成三层。v0.1、v0.2 是历史，以 v0.3 为准。

## Kun 的工作风格（重要——不照做会重蹈今天的错）

- **说人话，不用比喻。** 今天被否掉的比喻：「尺子」（→ 评分标准 / rubric）、「考题」（→ 测试集）、「旋钮」。用技术名。
- **被纠正时别过度更正。** 别一被纠就全盘推翻、又过度摆到另一头。找准那个稳的说法，别来回摆。
- **一切贴着真实实验来。** 别在抽象层凭空推。今天我多次因为「没回去看实物、在抽象里绕」被纠。
- **先对齐再执行**，别假设。别把回复压成 A/B 菜单。回复要紧，别注水。
